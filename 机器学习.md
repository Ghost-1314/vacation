# 一、概念
## 1.什么是机器学习？

- 定义：机器学习=让计算机**从数据中学习规律**，而不是靠人写死规则。

- 对比传统编程：
	- 传统编程：人写规则 + 输入数据 → 输出结果
	- 机器学习：输入数据 + 输出结果 → 机器**自己学会**规则

- 示例：
	- 传统：你写 `if 像素是黑 else 白` 来识别数字
	- 机器学习：给它 10 万张手写数字图片 → 它自己学会怎么认 0~9

- 总结：机器学习=用数据训练模型，让模型具备预测/分类能力。

---
## 2.机器学习和AI的关系


**人工智能（AI）**

　↓ 包含

**机器学习（ML）**

　↓ 包含

**深度学习（DL）**  


- 总结：

	- **AI 是一个大领域**：目标是让机器拥有智能（看、听、说、推理、决策）
	    
	- **机器学习是实现 AI 的一种方法**
	    
	- **深度学习是机器学习里最火、最强的一种方法**

- 形象比喻：

	- **AI = 整个汽车行业**
	    
	- **机器学习 = 燃油车**
	    
	- **深度学习 = 高性能电动车**

# 二、深度学习
## 1.定义

**深度学习**=用「**很深的神经网络**」来做机器学习。

- 关键：
	- 模型**层数很多**（“深度”）
	- 可以自动从数据里学特征
	- 适合图像、语音、文本这种复杂数据

总结：**深度学习=多层神经网络的机器学习**

---
## 2.最基础的神经网络

**三层**结构：
- **输入层 Input Layer**
- **隐藏层 Hidden Layer**
- **输出层 Output Layer**


### 1）**输入层 Input Layer**

- 作用：把数据喂给网络
- 示例：比如你要识别一张28×28的手写数字图片，那么：
	- 一共有784个像素
	- 每个像素是0~255的亮度
	- 输入层就有**784个神经元**

简单理解：
	**输入层=眼睛，负责“看数据”**



### 2）**隐藏层 Hidden Layer**

- 作用：**提取特征，学习规律**

- 这一层是人**看不见内部在干嘛**的，所以叫 “隐藏层”

- 隐藏层会做两件事情：

	1. 对输入做**加权求和**$$z=w_1​x_1​+w_2​x_2​+...+b$$
		- w = 权重（模型要学的东西）
		- b = 偏置（微调）
	
	2. 过一个激活函数（比如 ReLU）
		- 让模型能学会非线性规律


- 简单理解：**隐藏层=大脑，负责“思考、提取特征”**
- 如果有很多隐藏层，那就是深度神经网络，也就是深度学习


### 3）**输出层 Output Layer**

- 作用：给出最后结果
- 比如你要识别 0~9：
	- 输出层就 **10 个神经元**
	- 每个神经元对应一个数字的概率

输出层会告诉你：

- 是 0 的概率：1%
- 是 5 的概率：95%
- …

最后取概率最大的那个，就是模型的答案。

你可以理解：

**输出层 = 嘴巴，负责 “给出答案”。**


### 4）关系总结

- **输入层：传数据**
- **隐藏层：学规律**（层数越多越“深度”）
- **输出层：给结果**

流程（基础结构）：

	数据 → 输入层 → 隐藏层（学习） → 输出层（预测）

# 三、数据集
## 1）核心

- 数据集划分为训练集、验证集、测试集
- 训练集教模型做题，验证集帮你调参数，测试集最后判分
- 目的：让模型在没见过的新数据上也能用，不骗人



## 2）三个集合分别是什么&作用

### 1. 训练集 Training Set

- **作用：让模型学习规律**
- 模型在这里看数据、调权重、学特征
- 就像：**平时做作业、刷题**

你在这里做的事：

- 梯度下降
- 更新权重 W、b
- 让损失函数变小

---

### 2. 验证集 Validation Set

- **作用：调参、选模型，不参与训练**
- 训练时，模型**不看验证集的标签来学习**
- 每训练一轮，用验证集看效果
- 就像：**模拟考试，用来改学习方法**

你在这里做的事：

- 选学习率
- 选网络结构（几层、多少神经元）
- 早停（防止过拟合）
- 选最好的那个模型

**重点：验证集会被你 “反复用、反复看”，所以不能算真正的未知数据。**

---

### 3. 测试集 Test Set

- **作用：最终评估，模拟真实世界**
- **全程不参与训练，也不参与调参**
- 从头到尾只使用**一次**
- 就像：**高考，一考定终身**

意义：

- 告诉你模型**真正的泛化能力**
- 别人问你效果，你只报**测试集准确率**



## 3）为什么一定要分开？（关键）

如果不分开，只用一套数据，则：
- 模型会死记硬背（过拟合）
- 你会觉得效果超好，但是一上线真实数据就崩

所以，分开=防止**自欺欺人**



## 4）常见划分比例

- 数据少（几万以下）：**70% 训练 / 15% 验证 / 15% 测试**
- 数据多（百万级）：**98% 训练 / 1% 验证 / 1% 测试**


## 5）形象比喻

- 训练集：学生刷题（**学**）
- 验证集：老师随堂测，用来调整学习方法（**调**）
- 测试集：高考，完全密封，最后才拿出来（**考**）


# 四、神经网络

## 1.总括

- **神经网络 = 很多个神经元按层连在一起，用来拟合数据规律的数学模型**
- 结构：
	- 输入层
	- 隐藏层
	- 输出层


### 1）整体架构

1. **输入层（Input Layer）**

- 负责**接收数据**
- 神经元个数 = 你的**特征数量**
    
    - 比如一张图 28×28=784 像素 → 784 个神经元
    - 比如身高、体重、年龄 → 3 个神经元
    
- 只负责**传数据**，不做计算



2. 隐藏层（Hidden Layer）

- 神经网络的**大脑**
- 可以有**1 层、10 层、100 层**
    
    - 层数多 = 深度神经网络 = 深度学习
    
- 每一层都由一堆**神经元**组成
- 负责**提取特征、学习规律**



3. 输出层（Output Layer）

- 给出**最终结果**
- 神经元个数 = 你要预测的类别 / 数值数量
    - 二分类：1 个
    - 10 分类（0~9）：10 个
    - 回归预测：1 个

---
### 2）神经元是什么？（核心）

- 定义：**神经元 = 一个做「加权求和 + 激活」的数学单元**
- 神经元需要做的事：
	
	- **1）加权求和**
		
		输入：$$x_1​,x_2​,...,x_n​$$
		权重：$$w_1​,w_2​,...,w_n​$$
		偏置：$$b$$
		计算：$$z=w_1​x_1​+w_2​x_2​+...+w_n​x_n​+b$$
		- w 是**权重**：模型要学的重点
		- b 是**偏置**：让模型更灵活
	
	
	- **2）激活函数**
		
		把上面算出来的 z 塞进一个函数里：$$a=f(z)$$
		f 就是激活函数。常见的有：
		
		- `ReLU`（最常用）
		- `Sigmoid`（二分类）
		- `Tanh`

- **作用：让神经网络能学非线性、复杂规律。**

---
### 3）一个神经元的完整流程（重要）

1. 接收上一层传过来的**输入**

2. 每个输入乘上对应的**权重**

3. 全部加起来，再**加偏置 b**

4. 过**激活函数**

5. 把结果传给下一层

**这就是神经网络里最小的计算单元，也就是一个神经元的前向传播过程**

输入 → 加权和 + 偏置 → 激活 → 输出，写成公式就是$$a=f(w_1​x_1​+w_2​x_2​+⋯+w_n​x_n​+b)$$

---
### 4）神经元 → 层 → 网络

- 很多神经元排成一排 → **一层**

- 多层连起来 → **神经网络**

- 层数很多 → **深度神经网络 = 深度学习**

---

## 2.前向传播
### 1）什么是前向传播？

**前向传播 = 数据从输入层 → 隐藏层 → 输出层，一路往前算，最后得到预测结果**

**前向传播，即为网络的预测过程**

---

### 2）前向传播的完整流程

简单来说，从本质上说，前向传播就是从输入到输出，从头到尾算一遍：

即，输入数据 → 一层层算加权和 + 激活 → 最后输出预测值。

- 数据从**输入层进去**
- 经过**隐藏层一层层算**
- 从**输出层出来**
- 得到一个**预测结果**

**没有学习，没有更新，没有反向，只有 “往前算”。**

可以理解为：

- 模型已经**学完了**
- 权重 w、偏置 b 都**固定好了**
- 你丢进去一个 x
- 它**只往前算**，给你一个 y_hat（模型预测出来的结果）

所以，神经网络**通过前向传播来预测结果**

---
### 3）激活函数

- 定义：
	激活函数 = 给神经网络加“**非线性**”，让它能学复杂规律


- 如果没有激活函数：
	
	- 神经元做加权求和：$$z = w_1x_1 + w_2x_2 + … + b$$无论堆多少层隐藏层，最后数学上都等价于**一层线性模型**
	
	
	- 结论：**没有激活函数，网络只能学习直线，学不会复杂的东西**


- 激活函数做了什么？
	
	- 把算出来的 z ，做了一个非线性变换。公式：$$a=f(z)$$f 就是激活函数
	
	
	- 激活函数的作用：
		
		1. 引入**非线性**
		
		2. 让网络能拟合**任意复杂函数**（如图像、语音、文本等**复杂数据**全都能学）
		
		3. 控制信号要不要“激活”这个神经元



- 常用激活函数
	
	1.  ReLU（现在深度学习的绝对主流）$$f(z) = max(0, z)$$
		- 大于 0 就输出原值
		
		- 小于 0 就输出0
		
		- 计算快，好训练
		
		- **隐藏层默认用它**
		
		优点：快、稳、通用
		缺点：负数部分直接死了
	
	
	
	
	2. Sigmoid$$f (z) = 1/(1+e⁻ᶻ)$$输出 0~1 ，属于**二分类输出层**，缺点是容易**梯度消失**、计算慢，隐藏层不用
	
	
	
	
	3. Leaky ReLU（ReLU 小改进）$$LeakyReLU(z)=max(αz,z)$$
		- 负数时输出一点点，不至于完全死掉
		
		- 比 ReLU 更稳


---
### 4）神经元组成的网络是如何进行前向传播的

- 一层一层算过去，**每一层**里的每个神经元，都**独立做「加权和 + 激活」**

- **步骤**：
	
	- 第一步：输入层 → 隐藏层
	
		输入层把数据传给**隐藏层的每一个神经元**，隐藏层的每一个神经元都做**加权和**以及**过激活函数**，算完得到一整组输出称为**隐藏层输出**
	
	
	- 第二步：隐藏层 → 下一层隐藏层
		
		- 如果有多层，就重复上一步：
			
			- 上一层输出 → 这一层每个神经元的输入
			
			- 每个神经元：**加权和 + 激活**
	
	
	- 第三步：最后一层隐藏层 → 输出层
		
		输出层的每个神经元同样：
		
		1. **加权和**
		
		2. **激活**（根据任务选）
		
			- 二分类：Sigmoid
			
			- 多分类：Softmax
			
			- 回归：不用或线性
		
		最后得到输出结果



---
## 3.反向传播

### 1）定义

- **反向传播 = 算误差 → 从后往前传 → 改权重，让模型下次预测更准**

- 简单说，反向传播就是**神经网络“学习”的过程**

---

### 2）反向传播是用来干嘛的？

简单来说，前向传播算答案（预测），而反向传播则是**改作业（学习）**

**结果错了 → 算错多少 → 从输出层往回走 → 调整每一条连线的权重**

---
### 3）神经网络如何进行学习和训练？

- **流程**

	1. 先前向传播，算出预测
	
	2. 计算误差
	
	3. 反向传播开始：从后往前传误差
	
	4. 根据误差，**更新权重**，让下一次预测误差更小
	
	5. 前向传播 + 反向传播，**循环无数次** → 训练

---

### 4）损失函数

- 定义：
	
	**损失函数 = 衡量「预测值」和「真实值」差多少的函数**
	
	- 模型错得越厉害，损失越大；反之，模型越准，损失越小

---

- 损失函数在整个流程中的位置
	
	- 输入数据 → 前向传播 → 得到预测值 → 用==损失函数==，算错多少 → 反向函数，改权重
	
	- **没有损失函数，模型根本不知道自己错在哪**
	
	- 损失函数用来算误差大小，而反向传播就是靠这个误差去学习并修改权重


- 最常用的两种损失函数
	
	- 均方误差 MSE（回归问题用）
		
		预测**连续值**：房价、温度、分数等$$L(w,b) = \frac{1}{2m} \sum_{i=1}^m (\hat{y}_i - y_i)^2$$**差距越大，损失爆炸式变大**
		前面乘 1/2 是为了使得求导后更方便，不影响训练结果
	
	
	
	- 交叉熵 CrossEntropy（分类问题用）
		
		识别猫狗、数字、类别时用
		
		配合 Sigmoid / Softmax 一起用
		
		**分类任务默认必用**



- 损失函数的作用：
	
	- **给反向传播提供误差，告诉模型往哪改权重**
	
	- 流程：
		
		1. 前向传播 → 出答案
		
		2. 损失函数 → 算**错多少**
		
		3. 反向传播 → 根据误差**改权重**
		
		4. 循环 → 让损失越来越小

---

## 4.监督学习、无监督学习和半监督学习


- **监督学习**、**无监督学习**和**半监督学习**是机器学习最核心的三大流派。简单来说：
	
	**监督学习：有答案，照着学**
	
	**无监督学习：没答案，自己找规律**
	
	**半监督学习：少量答案 + 大量没答案，一起学**


### 1）监督学习（Supervised Learning）

- 核心：给模型【数据 + 正确答案】，让它学会映射

- 示例：
	
	- 图片 → 答案：是猫 / 是狗
	
	- 短信 → 答案：垃圾短信 / 正常短信
	
	- 面积、房间数 → 答案：房价
	
	- 题目 → 答案：分数
	
	机器在干嘛？它在根据输入学习输出
	
	也就是我们要告诉它对错，它被我们**监督着学习**

---
### 2）无监督学习（Unsupervised Learning）

- 核心：没有答案，没有标签，只有数据，让模型**自己**找结构，**自己**聚类、降维、找规律

- 示例：
	
	- 把用户分成：学生、上班族、老人（**聚类**）
	
	- 把很多特征压缩成少量关键特征（**降维**）
	
	- 找出哪些商品经常一起买（**关联规则**）

---
### 3）半监督学习（Semi-supervised Learning）

- 核心：少量有答案 + 大量没答案，**混合**训练

- 示例：
	
	- 有 1000 张图标了猫狗（有答案）
	
	- 有 100000 张图没标（没答案）
	
	- 用少量标签 + 大量无标签一起学

- 目的：省标注成本，效果比只用少量标签好。

---
简单来说：

- 监督学习，就是我们手把手全程教机器，完全由我们教

- 无监督学习，就是我们完全不教机器，也不给答案，让它自己学

- 半监督学习，就是我们先给机器点拨一点点，再让它自己举一反三


---

## 5.过拟合

- 形象解释：
	
	**过拟合 = 模型把训练题背下来了，但是不会做新题**

---
### 1）什么是过拟合？

- 在**训练集**上表现超级好，准确率几乎为**100%**（**训练准确率很高**）

- 在**测试集 / 新数据**上表现**特别差**（**测试准确率很低**）

- 只会做原题，不会做新题


原因：

- 模型并不是学会了规律，而是记住了训练数据里的**细节、噪音甚至错别字**

---
### 2）导致过拟合的原因？

1. **模型太复杂**
	
	网络层数太多、神经元太多，记忆力太强


2. **数据太少 / 太单一**
	
	见得少，容易死记硬背


3. **训练太久**
	
	本来已经学会规律了，还继续练，就开始背噪音

---
### 3）如何解决过拟合？

- **增加数据**

- **简化模型**（少几层、少神经元）

- **早停（Early Stopping）**：测试集变差就停

- **Dropout**：训练时随机关掉一些神经元

- **正则化**：惩罚太复杂的模型

---
## 6.数据归一化

### 1 ）作用

- 作用：
	1. **把所有特征缩到差不多的范围，让神经网络训练更快、更稳、更容易收敛**
	
	2. **让不同量级特征公平参与训练**
    
	3. **加快训练速度**
    
	4. **防止梯度消失/爆炸**
    
	5. **提高精度，避免不收敛**


- 为什么要这么做？
	
	- 比如一个特征：**年龄 0~100**
    
	- 另一个特征：**收入 0~1000000**
    
	收入数值**巨大**，年龄**很小**
	
	→ 模型会**偏心重视收入**，忽略年龄
	
	→ 训练**慢、难收敛、精度差**
	
	归一化后：
	
	所有特征都变成**差不多大小**
	
	→ 梯度下降走得直、走得快、不抖动
	
	→ 训练**又快又稳**

---
### 2）数据归一化最常用的方法

- Min-Max 归一化（归一化到 0~1）
	
	公式：$$x_{\text{norm}} = \frac{x - x_{\text{min}}}{x_{\text{max}} - x_{\text{min}}}$$
	结果：0 ~ 1
	适合：数据分布**没有特别极端值**时



- Z-Score 标准化（均值0，方差1，最常用）
	
	公式：$$x_{\text{std}} = \frac{x - \mu}{\sigma}$$ μ：均值
    
	σ：标准差
    
	结果：**大部分在 -1 ~ 1 之间**
    
	适合：**神经网络、深度学习默认首选**

---
## 7.正则化

### 1）作用？

- 正则化 = 给模型 “减负”，防止它太复杂、太用力，从而**避免过拟合**

- 可以简单理解成：**给模型加个 “约束”，不让它乱记噪音**

---
### 2）为什么要用正则化？

- 模型太复杂 → 容易**过拟合**

- 过拟合 = 背答案，不会做新题

所以，正则化，就是：**强制模型学得简单一点、平滑一点，不要死记硬背**

---
### 3）深度学习中常见的3种正则化

- ① L1 正则化

	
	- 让很多权重变成 **0**
	
	- 作用：**自动筛选特征，去掉没用的**



- ② L2 正则化（最常用）

	- 让权重**不要太大**
	
	- 作用：**让模型更平滑、更稳定**
	
	- 神经网络里最常用



- ③ Dropout（神经网络专属神器）

	- 训练时**随机关掉一部分神经元**
	
	- 不让模型依赖某几个神经元
	
	- 效果极强，几乎所有 CNN、Transformer 都用
---
## 8.数据清洗

- 概括：**数据清洗 = 把脏数据洗干净，让模型能够正常学习**

- **数据清洗 = 处理缺失、异常、重复、错误、格式混乱，让数据干净、规范、能用。**

- **机器学习里：数据清洗占约70% 工作量**


### 1）什么是脏数据？

- 脏数据，就是数据里的各种毛病：
	
	- 缺失值：有的字段是空的
	
	- 异常值：年龄写 1000、身高 5 米这种离谱数据
	
	- 重复数据：同一条记录出现好几次
	
	- 错误数据：格式不对、文字乱码、分类写错
	
	- 噪声：没用的、干扰模型的信息

- **脏数据 → 模型训练必崩**

- **垃圾进 → 垃圾出**

---
### 2）数据清理主要做的5件事

1. 处理缺失值

	- 删掉缺失太多的行 / 列
	
	- 用**均值、中位数、众数**填充
	
	- 用模型预测填充（高级）


2. 处理异常值（离群点）

	- 找出明显不合理的数据
	
	- 删掉或修正


3. 去重

	- 删掉完全重复的行


4. 格式统一

	- 把日期、文本、符号统一格式
	
	- 大小写统一、去掉多余空格


5. 数据转换（为建模做准备）

	- 文本转数字（如：男 / 女 → 0/1）
	
	- 类别特征编码
	
	- 数据归一化 / 标准化

## 9.**Batch Size** 和 **学习率**

**Batch Size** 和 **学习率** 是训练神经网络的两个核心超参数

- 定义：
	
	- **Batch Size**：每次喂给模型多少条数据来更新一次权重
	
	- **学习率（LR）**：每次更新权重时，步子迈多大
	
	这两个参数直接决定模型**能不能训好、训多快**，是调参的重中之重

### 1）Batch Size（批次大小）

- 核心逻辑：神经网络训练不是 “一条数据改一次权重” ，也不是 “所有数据都算完才改权重” ，而是把数据**分成若干批次**，**核心作用是平衡训练速度和稳定性**

- 用 1 个批次的数据跑完前向传播 → 算损失 → 反向传播 → 更新 1 次权重

- 所有**批次**都跑完 = 1 个 Epoch（一轮训练）

---
#### 3种典型取值

| 类型      | Batch Size取值 | 特点                 |
| ------- | ------------ | ------------------ |
| 批量梯度下降  | 等于全部数据       | 稳定，但数据量大时内存炸、训练极慢  |
| 小批次（主流） | 16/32/64/128 | 平衡速度和稳定性，是深度学习默认选择 |
| 随机梯度下降  | 1            | 快但抖动大，训练不稳定，几乎不用   |

---
#### 怎么选？

- 优先选 **32/64**（大部分场景通用）
    
- 显卡内存够大→选大一点（如128），训练更快
    
- 内存小→选小一点（如16），避免OOM（内存溢出）
    
- 数据少→Batch Size不能太大（否则等于批量梯度下降，易过拟合）
---
### 2）学习率

- 核心逻辑：反向传播算出“权重该怎么改”后，**学习率决定改的幅度**，是训练收敛的核心，默认0.001，需根据损失变化调整：$$\text{新权重} = \text{旧权重} - \text{学习率} × \text{梯度}$$

它就像你下山找最低点（最小损失）时的**步长**：

- 步子太大→跳过最低点，来回震荡，训不收敛
    
- 步子太小→走得太慢，训练耗时极长，还可能陷在局部最低点
    
- 步子刚好→快速走到最低点

---
#### 典型取值

- 深度学习默认：**1e-3（0.001）**
    
- 调优常用范围：1e-4 ~ 1e-2
    
- 极端情况：1e-5（保守）、1e-1（太激进，几乎不用）


---
#### 实用技巧

- **学习率衰减**：训练后期把学习率调小（比如每10轮减半），让模型精准收敛
    
- **试错法**：先从0.001开始，训不动（损失不降）就降到0.0001，震荡就再降
    
- 绝对别用太大的学习率（比如0.1），大概率直接训崩

# 五、多元线性回归

## 1）核心目标

你要找一组权重 $$w_1, w_2, ..., w_n$$ 和偏置 $$b$$，让多元线性回归模型 $$y = w_1x_1 + w_2x_2 + ... + w_nx_n +b $$ 的预测值尽可能接近真实值，而**梯度下降就是找这组参数的核心方法**。

---
## 2）理清核心公式


### （1）多元线性回归模型

假设有 n 个特征，模型表达式：$$\hat{y} = w_1x_1 + w_2x_2 + ... + w_nx_n + b = \sum_{i=1}^n w_i x_i + b$$其中：
- `y^`​：预测值
- `x1​,x2​...xn`​：特征（比如房价预测中的面积、房间数、楼层）
- `w1​,w2​...wn`​：特征权重
- `b`：偏置
---
### （2）损失函数（均方误差 MSE ）

我们要最小化“预测值和真实值的误差”，用 MSE 作为损失函数：$$L(w_1,w_2,...,w_n,b) = \frac{1}{2m} \sum_{i=1}^m (\hat{y}_i - y_i)^2$$其中：
- m：样本数量
- `y_i`​：第i个样本的真实值
- 乘以 1/2 ​是为了后续求导时抵消平方的系数，不影响最小值位置
---
### （3）梯度下降的核心逻辑

梯度是损失函数对每个参数的偏导数（可以理解为“参数往哪个方向改，损失会变小”），梯度下降的更新规则：$$\text{新参数} = \text{旧参数} - \text{学习率} × \text{损失对该参数的偏导数}$$

---
## 3）数学推导：求偏导数

我们以权重 $$w_j$$（第 j 个特征的权重）和偏置 $$b$$ 为例，求损失函数对它们的偏导数：


1. 对 w_j 的偏导数：$$\frac{\partial L}{\partial w_j} = \frac{1}{m} \sum_{i=1}^m (\hat{y}_i - y_i) x_{ij}$$`x_ij`​：第 i 个样本的第 j 个特征值


2. 对 b 的偏导数：$$\frac{\partial L}{\partial b} = \frac{1}{m} \sum_{i=1}^m (\hat{y}_i - y_i)$$

3. 参数更新公式
	
	最终得到梯度下降的参数更新规则：$$w_j = w_j - \alpha × \frac{1}{m} \sum_{i=1}^m (\hat{y}_i - y_i) x_{ij}$$$$b = b - \alpha × \frac{1}{m} \sum_{i=1}^m (\hat{y}_i - y_i)$$**α：学习率**

---
## 4）梯度下降的执行步骤


1. **初始化参数**：给所有 $$w_j$$ 赋随机小值（比如0.01）， b 赋0；
    
2. **前向计算**：用当前参数计算所有样本的预测值 $$\hat{y}$$；
    
3. **计算损失**：用 MSE 计算当前损失，判断是否收敛（损失不再下降）；
    
4. **反向求梯度**：计算损失对每个 $$w_j$$ 和 b 的偏导数；
    
5. **更新参数**：按上面的公式更新所有 $$w_j$$ 和 b；
    
6. **重复2-5步**：直到损失收敛到最小值。

---
## 5）实战代码

- 简单数据集（房价预测：面积、房间数 → 房价）演示：

```Python
import numpy as np

# 1. 准备数据（特征：面积、房间数；标签：房价）
# 样本数：4，特征数：2
X = np.array([[100, 2], [120, 3], [80, 1], [90, 2]])  # 特征矩阵 (m, n)
y = np.array([50, 65, 40, 45])                        # 真实房价 (m,)
m = len(X)  # 样本数
n = X.shape[1]  # 特征数

# 2. 数据归一化（必须！否则特征量级差异会导致训练失败）
X = (X - X.mean(axis=0)) / X.std(axis=0)

# 3. 初始化参数
w = np.random.randn(n)  # 权重 (n,)，随机小值
b = 0.0                 # 偏置
alpha = 0.01            # 学习率
epochs = 1000           # 训练轮数

# 4. 梯度下降核心循环
for epoch in range(epochs):
    # 前向计算：预测值
    y_hat = np.dot(X, w) + b  # (m,) = (m,n)·(n,) + (1,)
    
    # 计算损失（MSE）
    loss = np.sum((y_hat - y)**2) / (2 * m)
    
    # 反向求梯度
    dw = np.dot(X.T, (y_hat - y)) / m  # 对w的梯度 (n,)
    db = np.sum(y_hat - y) / m         # 对b的梯度
    
    # 更新参数
    w -= alpha * dw
    b -= alpha * db
    
    # 每100轮打印一次损失，看训练进度
    if epoch % 100 == 0:
        print(f"Epoch {epoch}, Loss: {loss:.4f}")

# 5. 测试模型
print("\n训练后的参数：")
print(f"权重 w (面积, 房间数)：{w.round(4)}")
print(f"偏置 b：{b.round(4)}")

# 预测新样本：面积归一化后≈0.5，房间数归一化后≈0.3
x_new = np.array([0.5, 0.3])
y_pred = np.dot(x_new, w) + b
print(f"\n新样本预测房价：{y_pred.round(2)} 万")
```

**关键注意事项：**

1. **数据归一化是刚需**：多元线性回归中特征量级差异大（比如面积是100，房间数是2），不归一化会导致梯度爆炸/消失，训练不收敛；
    
2. **学习率选择**：α太小（如 1e-5 ）训练慢，α 太大（如0.1）损失震荡不收敛，建议从0.01开始试；
    
3. **收敛判断**：不一定训满epochs，当损失连续多轮变化小于 1e-6 时，就可以停止训练（早停）；
    
4. **批量梯度下降**：上面的代码是“全批量”（用所有样本算梯度），实战中也可以用小批量（Batch）梯度下降（即你之前学的Batch Size）。

---
## 6）总结

1. 多元线性回归梯度下降的核心是**通过损失函数的梯度，迭代更新权重和偏置，直到损失最小**；
    
2. 关键步骤：初始化参数→前向计算→求梯度→更新参数→重复收敛；
    
3. 实战中必须做**数据归一化**，并合理选择学习率，才能让梯度下降稳定生效；
    
4. 代码实现时优先用矩阵运算（`numpy`），避免循环，提升效率。

# 六、多元线性回归的类封装

封装一个多元线性回归类，参考`sklearn`的接口风格（比如`fit()`、`predict()`方法），内部同时实现**解析解（正规方程）** 和**梯度下降法**两种求解参数的方式，且仅依赖`numpy`库

- 代码实现：

```Python
import numpy as np

class LinearRegression:
    """
    多元线性回归类（兼容解析解/梯度下降法）
    接口风格对齐sklearn，仅依赖numpy实现
    
    参数:
        solver : str, 可选（默认='normal_equation'）
            求解方法：
            - 'normal_equation': 解析解（正规方程）
            - 'gradient_descent': 梯度下降法
        learning_rate : float, 可选（默认=0.01）
            梯度下降的学习率（仅solver='gradient_descent'时生效）
        epochs : int, 可选（默认=1000）
            梯度下降的训练轮数（仅solver='gradient_descent'时生效）
        normalize : bool, 可选（默认=True）
            是否对特征做Z-Score标准化（仅梯度下降时建议开启）
    """
    def __init__(self, solver='normal_equation', learning_rate=0.01, epochs=1000, normalize=True):
        # 初始化超参数
        self.solver = solver
        self.learning_rate = learning_rate
        self.epochs = epochs
        self.normalize = normalize
        
        # 模型参数（训练后赋值）
        self.coef_ = None  # 权重 w1, w2, ..., wn (n,)
        self.intercept_ = None  # 偏置 b (1,)
        
        # 归一化相关参数（训练时记录，预测时复用）
        self._X_mean = None
        self._X_std = None

    def _normalize_X(self, X, is_train=True):
        """
        对特征矩阵做Z-Score标准化：X = (X - 均值) / 标准差
        is_train: 是否为训练阶段（训练阶段记录均值/标准差，预测阶段复用）
        """
        if not self.normalize:
            return X
        
        if is_train:
            self._X_mean = np.mean(X, axis=0)
            self._X_std = np.std(X, axis=0)
            # 防止标准差为0（特征全为同一值）导致除0
            self._X_std[self._X_std == 0] = 1.0
        
        # 预测阶段复用训练集的均值/标准差
        X_norm = (X - self._X_mean) / self._X_std
        return X_norm

    def fit(self, X, y):
        """
        训练模型（核心方法）
        参数:
            X : np.ndarray, 形状(m, n)
                训练特征矩阵（m个样本，n个特征）
            y : np.ndarray, 形状(m,)
                训练标签（真实值）
        返回:
            self : 当前实例
        """
        # 类型/形状校验
        X = np.array(X, dtype=np.float64)
        y = np.array(y, dtype=np.float64).reshape(-1, 1)  # 转为(m, 1)方便矩阵运算
        m, n = X.shape
        
        # 特征归一化（仅梯度下降时生效，解析解无需归一化）
        if self.solver == 'gradient_descent':
            X = self._normalize_X(X, is_train=True)
        
        # ================ 1. 解析解（正规方程）求解 ================
        if self.solver == 'normal_equation':
            # 构造增广矩阵：给X添加一列1（对应偏置b），形状(m, n+1)
            X_aug = np.hstack([np.ones((m, 1)), X])
            # 正规方程：θ = (X^T·X)^(-1)·X^T·y （θ包含b, w1, w2...wn）
            # 用np.linalg.inv求逆，np.linalg.pinv求伪逆（避免X^T·X不可逆）
            theta = np.linalg.pinv(X_aug.T @ X_aug) @ X_aug.T @ y
            # 拆分参数：theta[0]是b，theta[1:]是权重
            self.intercept_ = theta[0, 0]
            self.coef_ = theta[1:, 0]
        
        # ================ 2. 梯度下降法求解 ================
        elif self.solver == 'gradient_descent':
            # 初始化参数：权重随机小值，偏置为0
            self.coef_ = np.random.randn(n, 1)  # (n, 1)
            self.intercept_ = 0.0
            
            # 梯度下降核心循环
            for epoch in range(self.epochs):
                # 前向计算：预测值 (m, 1)
                y_hat = X @ self.coef_ + self.intercept_
                
                # 计算损失（MSE，可选：仅用于观察训练过程）
                loss = np.mean((y_hat - y) ** 2) / 2
                
                # 反向求梯度
                dw = (X.T @ (y_hat - y)) / m  # 权重梯度 (n, 1)
                db = np.mean(y_hat - y)       # 偏置梯度
                
                # 更新参数
                self.coef_ -= self.learning_rate * dw
                self.intercept_ -= self.learning_rate * db
                
                # 每100轮打印损失（可选）
                if epoch % 100 == 0:
                    print(f"Epoch {epoch}, Loss: {loss:.6f}")
            
            # 把权重从(n,1)转为(n,)，对齐sklearn格式
            self.coef_ = self.coef_.reshape(-1)
        
        else:
            raise ValueError(f"不支持的求解器：{self.solver}，仅支持 'normal_equation' 或 'gradient_descent'")
        
        return self

    def predict(self, X):
        """
        预测（核心方法）
        参数:
            X : np.ndarray, 形状(m, n)
                待预测的特征矩阵
        返回:
            y_pred : np.ndarray, 形状(m,)
                预测值
        """
        # 校验模型是否已训练
        if self.coef_ is None or self.intercept_ is None:
            raise RuntimeError("请先调用fit()训练模型")
        
        # 类型转换 + 归一化（复用训练集的均值/标准差）
        X = np.array(X, dtype=np.float64)
        if self.solver == 'gradient_descent':
            X = self._normalize_X(X, is_train=False)
        
        # 预测计算：y = w1x1 + w2x2 + ... + wnxn + b
        y_pred = X @ self.coef_ + self.intercept_
        return y_pred

    def score(self, X, y):
        """
        计算决定系数R²（评估模型效果，对齐sklearn）
        R²越接近1，模型拟合效果越好
        """
        y_pred = self.predict(X)
        ss_total = np.sum((y - np.mean(y)) ** 2)  # 总平方和
        ss_residual = np.sum((y - y_pred) ** 2)   # 残差平方和
        r2 = 1 - (ss_residual / ss_total)
        return r2


# ====================== 测试代码 ======================
if __name__ == "__main__":
    # 1. 准备测试数据（房价预测：面积、房间数 → 房价）

    y_train = np.array([50, 65, 40, 45, 55, 70])

    # 3. 测试梯度下降法求解
    print("===== 梯度下降法求解 =====")
    lr_gd = LinearRegression(solver='gradient_descent', learning_rate=0.01, epochs=2000)
    lr_gd.fit(X_train, y_train)
    print(f"权重 coef_: {lr_gd.coef_}")
    print(f"偏置 intercept_: {lr_gd.intercept_}")
    y_pred_gd = lr_gd.predict(X_test)
    print(f"测试集预测值: {y_pred_gd}")
    print(f"R²评分: {lr_gd.score(X_test, y_test):.4f}")
```

# 七、K-means算法

- K-means 算法是**无监督学习中最核心、最常用的聚类算法**

## 1）定义

K-means = **把一堆无标签的数据，自动分成 K 个 “相似的组”（簇）**，核心是**让每个组内的数据尽可能像，组间的数据尽可能不像**

形象比喻（老师分座位）：

把K-means比作“老师分座位”：

- 全班同学（数据）没有任何标签，老师要把他们分成K组（比如3组）；
    
- 第一步：随便选3个同学当“组长”（初始聚类中心）；
    
- 第二步：所有同学根据“和组长的相似度”坐到对应组（分配簇）；
    
- 第三步：每个组重新选“最中间的同学”当新组长（更新聚类中心）；
    
- 重复二、三步，直到组长位置不再变（收敛）。

---
## 2）K-means 算法的完整流程

- 前置条件
	
	- 输入：无标签数据集 X ，（形状 m × n ，m 个样本，n 个特征），预设簇数 K ；
	
	- 输出：每个样本的簇标签，K 个聚类中心。

## 3）步骤拆解

**步骤1：初始化聚类中心**

- 核心：从所有样本中**随机选 K 个**作为初始中心（形状 K × n）；

- 注意：初始中心的选择会影响最终聚类结果（可能陷入局部最优），可多次运行选最优结果。



**步骤2：计算距离（核心是欧氏距离）**

对每个样本 x_i 和每个聚类中心 c_j ，计算欧式距离：$$d(x_i, c_j) = \sqrt{\sum_{k=1}^n (x_{ik} - c_{jk})^2}$$
- x_ik：第 i 个样本的第 k 个特征：

- c_jk：第 j 个聚类中心的第 k 个特征；

- 本质：衡量样本和中心的“相似度”，距离越小越相似。



**步骤3：分配簇标签**

- 对每个样本，找到距离最小的聚类中心，把该样本的标签设为这个簇的编号。



**步骤4：更新聚类中心**

对每个簇，计算所有样本的**特征均值**，作为新的聚类中心：$$c_j = \frac{1}{m_j} \sum_{x_i \in 簇j} x_i$$
- m_j：第 j 个簇的样本数量；

- 本质：让新中心落在簇的“几何中心”，保证簇内的样本最相似。



**步骤5：判断收敛**

- 收敛条件：新聚类中心和旧中心的**误差小于阈值**（比如0.001），或达到最大迭代次数；

- 误差计算：所有 K 哥中心的欧氏距离之和，若几乎为0，说明中心不再动，聚类稳定。

## 4）K-means 实战代码（纯`numpy`实现，对齐`sklearn`接口）

- AI生成

```Python
import numpy as np

class KMeans:
    """
    K-means聚类算法（纯numpy实现，对齐sklearn接口）
    参数:
        n_clusters : int, 可选（默认=3）
            要划分的簇数K
        max_iter : int, 可选（默认=300）
            最大迭代次数
        tol : float, 可选（默认=1e-4）
            收敛阈值（中心变化小于该值则停止）
        random_state : int, 可选（默认=42）
            随机种子（保证结果可复现）
    """
    def __init__(self, n_clusters=3, max_iter=300, tol=1e-4, random_state=42):
        self.n_clusters = n_clusters
        self.max_iter = max_iter
        self.tol = tol
        self.random_state = random_state
        
        # 训练后生成的参数（带下划线）
        self.cluster_centers_ = None  # K个聚类中心 (K, n)
        self.labels_ = None           # 每个样本的簇标签 (m,)

    def _euclidean_distance(self, x1, x2):
        """计算两个向量的欧氏距离"""
        return np.sqrt(np.sum((x1 - x2) ** 2, axis=1))

    def fit(self, X):
        """
        训练K-means模型
        参数:
            X : np.ndarray, 形状(m, n)
                无标签特征矩阵
        """
        X = np.array(X, dtype=np.float64)
        m, n = X.shape
        
        # ========== 步骤1：初始化聚类中心 ==========
        np.random.seed(self.random_state)
        # 随机选K个样本作为初始中心
        init_idx = np.random.choice(m, self.n_clusters, replace=False)
        self.cluster_centers_ = X[init_idx].copy()
        
        # 迭代更新
        for _ in range(self.max_iter):
            # ========== 步骤2+3：计算距离 + 分配簇标签 ==========
            # 对每个样本，计算到K个中心的距离 (m, K)
            distances = np.zeros((m, self.n_clusters))
            for j in range(self.n_clusters):
                distances[:, j] = self._euclidean_distance(X, self.cluster_centers_[j])
            # 分配标签：每个样本选距离最小的簇 (m,)
            self.labels_ = np.argmin(distances, axis=1)
            
            # ========== 步骤4：更新聚类中心 ==========
            new_centers = np.zeros((self.n_clusters, n))
            for j in range(self.n_clusters):
                # 找到当前簇的所有样本
                cluster_samples = X[self.labels_ == j]
                if len(cluster_samples) > 0:
                    new_centers[j] = np.mean(cluster_samples, axis=0)
                else:
                    # 极端情况：簇内无样本，重新随机选一个样本当中心
                    new_centers[j] = X[np.random.choice(m, 1)]
            
            # ========== 步骤5：判断收敛 ==========
            # 计算新旧中心的误差（所有中心的距离之和）
            center_diff = np.sum(self._euclidean_distance(new_centers, self.cluster_centers_))
            if center_diff < self.tol:
                break  # 收敛，停止迭代
            
            # 更新中心
            self.cluster_centers_ = new_centers
        
        return self

    def predict(self, X):
        """
        预测新样本的簇标签
        参数:
            X : np.ndarray, 形状(m, n)
                待预测特征矩阵
        返回:
            labels : np.ndarray, 形状(m,)
                每个样本的簇标签
        """
        if self.cluster_centers_ is None:
            raise RuntimeError("请先调用fit()训练模型")
        
        X = np.array(X, dtype=np.float64)
        distances = np.zeros((len(X), self.n_clusters))
        for j in range(self.n_clusters):
            distances[:, j] = self._euclidean_distance(X, self.cluster_centers_[j])
        return np.argmin(distances, axis=1)
```

## 5）注意事项

1. **必须做特征归一化**：和梯度下降一样，K-means基于距离计算，特征量级差异会导致聚类结果偏向大数特征（比如“收入”远大于“年龄”，聚类会只看收入）；
    
2. **K值的选择是核心难点**：
    
    -  没有绝对正确的K值，常用“肘部法则”（看聚类误差随K增加的变化，找“拐点”）；
        
    - 比如K=2时误差很大，K=3时误差骤降，K>3时误差下降缓慢，说明K=3是最优；
        
3. **初始中心的影响**：随机初始中心可能导致局部最优，实战中可多次运行（设置不同随机种子），选聚类误差最小的结果；
    
4. **适用于“凸形簇”**：K-means对圆形/椭圆形簇效果好，对不规则形状（比如月牙形）效果差（可换`DBSCAN`算法）。

## 6）总结

1. K-means核心流程：**初始化中心→计算距离→分配簇→更新中心→判断收敛**，循环至稳定；
    
2. 核心计算：欧氏距离（衡量相似度）、均值（更新中心）；
    
3. 关键操作：特征归一化、合理选择K值、多次运行避免局部最优；
    
4. 代码实现：对齐`sklearn`接口（`fit()`训练、`predict()`预测），仅依赖`numpy`即可完成。
